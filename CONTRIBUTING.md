# Contributing to Đuka Protocol

Thank you for considering contributing to Đuka!

## Code of Conduct
We follow the [Contributor Covenant](https://www.contributor-covenant.org/). Be respectful, professional, and focused on building safe, unified intelligence systems.

## Development Setup
1. Fork the repo
2. `docker-compose up -d`
3. Work on a feature branch (`git checkout -b feature/amazing-feature`)
4. Run existing tests and add new ones
5. Ensure code passes linting

## Security Requirements for Contributions
- Never commit secrets, API keys, or sensitive data.
- All PRs touching security-related files (Firewall, Guardrail, DP-FedAvg, etc.) will receive extra review.
- We require **CLA** (Contributor License Agreement) for significant contributions (to be added soon).

## Pull Request Process
1. Open an Issue first for big changes.
2. Make sure your PR includes:
   - Clear description
   - Tests
   - Updated documentation
   - Security impact assessment (if relevant)
3. Branch protection: All PRs need at least one approval + passing CI (to be added).

## Areas Where We Need Help

## Enterprise & Big Tech Focus
